# HANDOFF — claude.ai 대화 인수인계 (2026-06-12)

이 프로젝트는 claude.ai 채팅에서 바이브코딩으로 만들어졌고, 이 문서는 그 대화의 결정사항 전체 기록이다.

## 대화 타임라인

1. **시작**: "바이브코딩 해줘" → 사용자가 직접 기획안 제시:
   - 주식 현재가 적정여부 판단 (PER, 재무제표 영업이익, DART 공시, 최근 이슈)
   - 개인 투자 가치관 기반 손절/익절 가이드 (3~5단계 유형 진단 → 맞춤 매도 가이드)
   - 국내외 이슈 크롤링·호재악재·섹터·애널리스트 요약
   - 모든 내용을 쉬운 눈높이로 풀 것
2. **앱 이름 확정**: 〈주식도 AI〉
3. **v1 (claude.ai 아티팩트)**: React 단일 파일. Claude API + web_search 도구로 데이터를 '검색·요약'하는 구조. 한계: 실시간 아님, 환각 가능, 공시 빈약.
4. **눈높이 변경**: 중학생 → **고등학생** (용어는 쓰되 첫 등장 시 괄호 풀이)
5. **이슈 종목 탭 추가**: 검색 없이 화제 종목의 현재가·이슈·거래량 자동 정리. 카드 클릭 → 분석 탭 자동 이동.
6. **구조 설명 요청**: 데이터가 'AI 웹검색 요약'임을 명확화. 환각·시점불일치·파싱깨짐 등 이슈 정리.
7. **진짜 데이터 전환 결정**: DART OpenAPI(공식) + FinanceDataReader/pykrx(시세·PER) 조사. CORS·키노출·샌드박스 때문에 **백엔드 필요** 결론.
8. **언어 선택**: 파이썬 vs JS 비교 → 한국 주식 라이브러리 생태계(FDR·pykrx·OpenDartReader 전부 파이썬) 때문에 **파이썬(FastAPI)** 채택.
9. **DART 키 수령** → 백엔드 구축. `.env` 분리. ⚠️ 키가 채팅에 평문 노출됐으므로 재발급 권고됨.
10. **프론트엔드 분리 구축** (React+Vite, /api 프록시). 이슈 종목도 KRX 실데이터(거래대금 상위·급등·급락)로 전환.
11. **화면 미리보기** 제공 (인터랙티브 목업).
12. **노트 라벨**: '주식도AI 한마디'로 정리(폰트 Pretendard 통일).
13. **테스트 준비**: 원클릭 실행 스크립트(start-*.bat/sh) 추가. 백엔드 부팅·헬스체크·테스트페이지 200 OK 검증 완료 (단, 그 환경은 외부망 차단이라 실데이터 미확인).
14. **클로드코드 이전**: 본 문서 + CLAUDE.md 작성.

## 핵심 설계 결정과 이유

- **하이브리드 구조**: 숫자는 API(DART·KRX)에서 진짜 값, AI는 '설명만' 담당 → 환각으로 가격 틀릴 가능성 제거. explain.py 프롬프트에 "숫자를 절대 바꾸지 마" 명시.
- **죽지 않는 백엔드**: 모든 외부 호출 try/except → 빈 값 반환. 외부망 없는 환경에서도 200 응답 확인됨.
- **숫자 먼저, 설명 나중**: Analyzer가 데이터를 먼저 렌더하고 /api/explain 을 비동기로 따로 호출.
- **투자유형 진단은 로컬 로직**: 5문항(각 1~3점) 합산 5~15점 → 🛡️안전제일(5-7)/⚖️균형(8-10)/🚀도전(11-12)/🔥하이리스크하이리턴(13-15). 가이드 텍스트는 quizData.js에 하드코딩.

## 검증 상태
- 백엔드: 문법·임포트·라우트·키로드·부팅·/api/health 200 ✅ / 실데이터 호출 ❌(환경 제약, 사용자 PC에서 첫 확인 필요)
- 프론트: JS 문법·import 경로·괄호균형 ✅ / `npm run build` ❌(미실행 — 첫 우선 점검 대상)

## 미해결 / 주의
- DART 키 재발급 필요 가능성 (채팅 노출)
- 사용자는 비개발자 성향 — 설명은 쉽게, 실행은 스크립트 중심으로
